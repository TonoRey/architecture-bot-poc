Architecture Decisions — PoC
01
Google Colab as the platform — Zero infra setup. The entire PoC runs in a browser-based notebook. Easy to demo live, easy to share with the team via Drive link. No Cloud Run, no GKE for the PoC.
02
Gemini 1.5 Flash as primary LLM — Free tier (1M tokens/day), low latency, natively integrated with GCP ecosystem. OSS fallback: Groq free tier with Llama 3.3 70B if quota runs out.
03
NetworkX over Spanner Graph — In-memory graph for PoC. Zero cost, zero setup. If the boss approves the project, we migrate to Spanner Graph or Neo4j for production.
04
FAISS over Vertex AI Vector Search — In-memory vector store for PoC. Runs locally in Colab. Production path: Vertex AI Vector Search when scaling to multiple diagrams.
05
Draw.io XML as single source of truth — The richer the labels and metadata in the diagram, the smarter the chatbot. Before Sprint 1, Antonio enriches the diagram with component types, tags, and descriptions.
06
Learning by doing with Claude — No external courses. Every line of code is built together in this conversation. The team learns by reading, questioning, and modifying working code — not watching videos.
